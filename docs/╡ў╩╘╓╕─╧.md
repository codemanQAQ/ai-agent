# 本地调试指南

本文档记录在 macOS 上调试 `ai-agent` 服务所需的环境、版本与启动步骤，确保团队成员可在本机一键复现开发环境。

## 1. 主机环境（参考基线）

| 项 | 版本 |
| --- | --- |
| 操作系统 | macOS 26.5 (Build 25F71) |
| 架构 | Apple Silicon `arm64` (aarch64) |
| Shell | zsh |
| Git | 2.50.1 (Apple Git-155) |

> 团队成员若使用 Intel Mac (x86_64) 或 Linux 也可，但需自行验证 Milvus / RocketMQ 客户端的原生依赖是否齐全。

## 2. 运行时与构建工具

| 工具 | 版本 | 说明 |
| --- | --- | --- |
| JDK | **Oracle GraalVM 25.0.2+10.1 (LTS)** | 项目要求 `java.version=25`、`maven.compiler.release=25`，必须使用 JDK 25。GraalVM 用于后续 Native Image 编译。 |
| Maven | 3.9.14（项目内置 `./mvnw` 一致版本） | 推荐用 `./mvnw` 而非全局 mvn，保证版本一致。 |
| Node.js | v22.22.0 | 可选；仅在前端 / 工具脚本时需要。 |
| SDKMAN | 推荐 | 用于切换 JDK：`sdk install java 25.0.2-graal && sdk use java 25.0.2-graal`。 |

JDK 校验：

```bash
java -version
# 期望输出包含：openjdk version "25" 或 GraalVM 25.0.2
./mvnw -v
# 期望 Java version: 25.0.2
```

> 全部使用云端 / 远端服务，无需本机部署。配置见 `src/main/resources/application-dev.properties`，敏感值在 `.env.example` 中以占位符示意。

| 组件 | 用途 | 接入方式 |
| --- | --- | --- |
| PostgreSQL **18.x** (Neon Serverless) | 业务库 / Outbox 持久化 | `spring.datasource.url=jdbc:postgresql://...neon.tech/involutionhell?sslmode=require` |
| Milvus / Zilliz Serverless | 向量库 | `rag.milvus.uri=https://...zilliz.com` + token |
| RocketMQ **5.x** (Proxy gRPC) | 离线索引消息队列 | `rag.rocketmq.endpoints=35.201.24.89:8081`（dev 默认值），客户端 SDK `rocketmq-client-java 5.x` |
| Elasticsearch **8.16.6** | Elastic APM 后端存储 / 日志检索 | 由 APM Server 写入；服务自身不直连 ES |

| Elastic APM (OTLP) | 链路 / 指标 / 日志上报 | OpenTelemetry Java Agent → `apm-server.interview.svc.cluster.local:8200` |

## 5. 启动方式

### 5.1 命令行启动（dev profile）

```bash
./mvnw spring-boot:run -Dspring-boot.run.profiles=dev
```

> 默认端口 `8080`，contextPath `/api/v1`，健康检查：`curl http://localhost:8080/api/v1/actuator/health`

### 5.2 IntelliJ IDEA Run Configuration

仓库已包含 `.run/RagApplication [dev].run.xml`：

- 主类：`com.bytedance.ai.RagApplication`
- Module：`ai-agent`
- Active profiles：`dev`
- 已附加 OpenTelemetry Java Agent + 全部 `-Dotel.*` 参数，但**本地默认 `-Dotel.sdk.disabled=true`** —— 因为 `apm-server.interview.svc.cluster.local` 只在 k8s 集群内可解析，本机直连必报 `UnknownHostException`
- 需要本地链路调试时：把 `-Dotel.sdk.disabled=true` 改为 `false`，并通过 `kubectl port-forward svc/apm-server 8200:8200 -n interview` 或 IDEA k8s 隧道映射，把 `-Dotel.exporter.otlp.endpoint` 改成 `http://localhost:8200`

IDEA 中右键 `RagApplication [dev]` → Debug 即可断点调试。

### 5.3 VM 参数清单（与 `.run/*.run.xml` 保持一致）

```text
-javaagent:$PROJECT_DIR$/elastic-otel-javaagent.jar
-Dotel.metric.export.interval=10000
-Dotel.exporter.otlp.endpoint=http://apm-server.interview.svc.cluster.local:8200
-Dotel.exporter.otlp.headers=Authorization=Bearer%20<TOKEN>
-Dotel.exporter.otlp.protocol=grpc
-Dotel.service.name=rag-backend-dev
-Dotel.resource.attributes=deployment.environment=dev,service.namespace=rag
-Dotel.traces.exporter=otlp
-Dotel.metrics.exporter=otlp
-Dotel.logs.exporter=otlp
-Dotel.instrumentation.micrometer.enabled=true
-Dotel.instrumentation.runtime-telemetry.enabled=true
-Dotel.instrumentation.runtime-telemetry.emit-experimental-metrics=true
-Dotel.instrumentation.runtime-telemetry.emit-experimental-telemetry=true
-Dotel.instrumentation.http.server.emit-experimental-telemetry=true
-Dotel.instrumentation.servlet.experimental.capture-request-parameters=*
-Dotel.traces.sampler=always_on
-Dotel.instrumentation.rocketmq-client.enabled=true
--enable-native-access=ALL-UNNAMED
--sun-misc-unsafe-memory-access=allow
```

- `--enable-native-access=ALL-UNNAMED` / `--sun-misc-unsafe-memory-access=allow`：JDK 25 收紧了原生访问检查，gRPC / Netty / Milvus SDK 仍依赖 `sun.misc.Unsafe`，未加这两项会启动报警告甚至失败。
- `traces.sampler=always_on` 仅 dev 用，生产请改为 `parentbased_traceidratio` + 概率采样。

## 6. 常见问题排查

| 现象 | 原因 | 解决 |
| --- | --- | --- |
| `UnsupportedClassVersionError` | 使用了 JDK < 25 | `sdk use java 25.0.2-graal` |
| 启动时 `WARNING: A restricted method in java.lang.System has been called` | JDK 25 原生访问限制 | 确认 VM 参数包含 `--enable-native-access=ALL-UNNAMED` |
| Milvus 调用报 `gRPC routing` 异常 | `rag.milvus.database-name` 非空 | Zilliz Serverless 必须留空 |
| RocketMQ 客户端连接超时 | dev endpoint `35.201.24.89:8081` 不通 | 检查 VPN / 网络；或临时 `rag.rocketmq.enabled=false` 走同步索引 |
| 邮件发送 535 认证失败 | Gmail 普通密码无效 | 使用 [App Password](https://myaccount.google.com/apppasswords) |
| OTLP 上报无数据 | APM endpoint 不可达 / Bearer Token 失效 | 检查 `apm-server.interview.svc.cluster.local` 是否在集群网内；本机调试可临时去掉 `-javaagent` |
| `port 8080 already in use` | 端口冲突 | `lsof -i:8080` 找到进程；或 `-Dserver.port=8081` |

## 7. 调试技巧

- **本地断点调试**：IDEA Debug 模式启动 `RagApplication [dev]` 即可。
- **远程调试（推荐：IDEA 自带 Kubernetes 隧道）**：
  1. 在 IDEA 中打开 **Services 工具窗** → 连接 `kubeconfig`（或直接使用 K8s 插件加载 context）。
  2. 选中目标 Pod（如 `rag-backend-dev-xxx`）→ 右键 **Forward Ports to Pod...**，将远端 JVM 调试端口（默认 `5005`）映射到本地。
  3. IDEA 新建 **Run/Debug Configuration → Remote JVM Debug**，host 填 `localhost`、port 填映射后的本地端口，**Attach**。
  4. 远端容器需以 debug 模式启动：在部署 manifest 的 JVM 参数追加 `-agentlib:jdwp=transport=dt_socket,server=y,suspend=n,address=*:5005`（dev 环境已默认开启，prod 不要开）。
  5. 无需再用 `kubectl port-forward` 手动起隧道，IDEA 会在 Debug 会话期间维持端口转发，断开自动回收。
- **查看 Actuator**：`/api/v1/actuator/health`、`/metrics`、`/loggers`（dev 已暴露）。
- **动态调日志级别**：
  ```bash
  curl -X POST http://localhost:8080/api/v1/actuator/loggers/com.bytedance.ai \
       -H 'Content-Type: application/json' \
       -d '{"configuredLevel":"DEBUG"}'
  ```
- **Modulith 模块边界校验**：`./mvnw -Dtest=ApplicationModulesVerificationTests test`。
- **本地 OTel 关闭**：临时注释 `-javaagent` 行即可，不影响功能。
