# 索引链路运行手册（W1-IDX-05 验收）

> 把 catalog 商品数据从 JSON 一路索引到 Milvus，端到端 6 步走通。
> 本文档既是 demo 现场操作脚本，也是 oncall 排查链路的速查表。

---

## 0. 前置依赖

| 组件 | 用途 | 启动建议 |
|---|---|---|
| PostgreSQL 16+ | rag_documents / rag_chunks / catalog_spu / catalog_sku 主存 | 本地 Docker 或 Neon |
| Milvus 2.4+ | 向量索引（含 IMAGE chunk） | 本地 docker compose / Zilliz Cloud |
| RocketMQ 5.x | indexing pipeline outbox 消息（可关掉用 in-process） | 本地 dashboard 或 ACL |
| Doubao（火山方舟） | ChatModel + Embedding（建议 Doubao-embedding-vision） | API Key 已就绪 |

`.env`（参考 `.env.example`）至少配齐：

```properties
spring.datasource.url=jdbc:postgresql://...
spring.ai.openai.api-key=ark-xxx
spring.ai.openai.base-url=https://ark.cn-beijing.volces.com/api/v3
spring.ai.openai.embedding.model=doubao-embedding-vision-250328
spring.ai.openai.embedding.dimensions=2048
rag.milvus.enabled=true
rag.milvus.uri=http://localhost:19530
rag.milvus.embedding-dimension=2048    # ⚠️ 必须与 embedding.dimensions 一致
rag.rocketmq.enabled=true              # 不启用时索引仍可串行跑通，但缺少 outbox 演示价值
rag.catalog.enabled=true
```

## 3. 6 步手动验收清单

如果脚本不便使用（譬如线上 oncall），按这 6 步逐个核对：

| # | 检查 | SQL / 命令 | 预期 |
|---|---|---|---|
| 1 | 主数据 | `SELECT count(*) FROM catalog_spu;` | 10 |
| 2 | SKU 数量 | `SELECT count(*) FROM catalog_sku;` | ≥ 10 |
| 3 | 双写到 rag_documents | `SELECT count(*) FROM rag_documents WHERE source_type='catalog-spu';` | 10 |
| 4 | document_id 全部回填 | `SELECT count(*) FROM catalog_spu WHERE document_id IS NULL;` | 0 |
| 5 | 索引完成 | `SELECT status, count(*) FROM rag_documents WHERE source_type='catalog-spu' GROUP BY status;` | 全部 INDEXED |
| 6 | chunk_type 分类正确 | `SELECT metadata->>'chunkType' AS t, count(*) FROM rag_chunks c JOIN rag_documents d ON d.id=c.document_id WHERE d.source_type='catalog-spu' GROUP BY 1;` | TITLE / DESC / ATTR 至少各 ≥ 8 行 |
| 7 | 属性抽取成功 | `SELECT attributes_status, count(*) FROM catalog_spu GROUP BY 1;` | DONE ≈ 10（少数 FAILED 可接受） |
| 8 | Milvus 命中 | `curl -X POST $BASE_URL/public/rag/ask -d '{"query":"轻量通勤双肩包"}'` | 答案 contexts 中 metadata.source_type='catalog-spu' |

## 4. 故障速查

| 现象 | 可能原因 | 排查 |
|---|---|---|
| `rag_documents.status` 卡在 PENDING | RocketMQ 未启动 / outbox 未派发 | `SELECT * FROM rag_index_outbox WHERE status='PENDING'`；查 `RagOutboxDispatcher` 日志 |
| `status=FAILED, last_error=...dimension mismatch...` | embedding.dimensions ≠ Milvus collection dimension | 改齐 `.env`，重建 Milvus collection |
| `catalog_spu.attributes_status` 全部 FAILED | LLM 输出非纯 JSON 或速率超限 | 看 `LlmAttributeExtractor` WARN 日志；调 `rag.catalog.extraction-concurrency` 降并发 |
| `catalog_spu.attributes_status` 全部 PENDING | `rag.catalog.enabled=false` 或 worker executor 未启动 | 查启动期 `RagConcurrencyConfiguration` 日志 |
| 文本可搜到，图片搜不到 | chunk_type=IMAGE 还没接入（W3 任务） | W1 不涉及，先确认文本路径打通 |

---

## 5. 与既有 markdown 文档共存

- catalog 导入走 `source_type='catalog-spu'`，与既有 `source_type='markdown'` 文档在同一 `rag_documents` / `rag_chunks` 表中混合存放。
- chunk_type 分类只对 catalog-spu 启发式生效，markdown 文档统一归 `BODY`，**对既有索引零影响**。
- 检索阶段未来按 chunk_type 加权时，BODY 默认走中性权重；TITLE / ATTR 命中加成更高。
