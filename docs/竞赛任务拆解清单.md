# 竞赛任务拆解清单（服务端）

> 配套文件：`竞赛技术方案规划.md`。本清单把方案拆成可勾选的工作项，按 4 周里程碑组织。
> 颗粒度原则：每条 0.5–1 天可完成；带 ⭐ 的是评分加分关键项；带 🔗 的是与客户端联调依赖项。

## 任务编号规则

```
[W{周}-{模块}-{序号}] 标题
模块：CAT=catalog | AGT=agent | RET=retrieval | CART=cart | ORD=order
     IDX=indexing | INF=infrastructure | CLI=客户端协议 | EVAL=回归评估 | OPS=运维交付
```

## 优先级提示（如果时间紧张，砍这些）

按"评分性价比"从低到高排序，紧急时按顺序砍：

1. **场景化组合推荐**（`SCENARIO_BUNDLE`）— 加分项之一，但工作量大于产出，可砍
2. GraalVM Native 镜像 — 锦上添花，启动时间不是主评分点
3. `ConversationSummarizer` 长上下文摘要 — 多轮 6 轮内基本不漂移
4. RRF 融合 — 单路 vision 检索效果已经够好的话可砍
5. 价格变动二次确认 — 演示数据稳定的话可砍

**永远不砍**：基础场景 2 个 / 反选 / 购物车主流程 / 拍照找货 / 引用校验 / 回归集。
